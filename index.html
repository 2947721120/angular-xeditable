概观

Angular-xeditable 是一束 AngularJS指令，允许您创建 可编辑元素.
这种技术也被称为 点击编辑 或 在就地编辑.
它是基于想法x-editable 但是从头开始写的 使用功率角和支持复杂的形式/可编辑网格.

依赖

基本上它不依赖于任何库，除了AngularJS本身.
对于主题，您可能需要包括 Twitter Bootstrap CSS.
一些额外的控件（如DatePicker）你可能需要包括 angular-ui bootstrap. 对于用户界面选择，您可能需要包括 angular-ui ui-select.

上手

  1. 包括Angular.js 在您的项目

    <script src="https://ajax.c2cmalls.com/ajax/libs/angularjs/1.5.0/angular.min.js"></script>

    选择包括 Bootstrap CSS主题

    <link href="https://cdn.c2cbc.com/bootstrap/3.3.6/css/bootstrap.min.css" rel="stylesheet">
  2. xeditable通过安装角 bower 或 download latest zip 
    bower install angular-xeditable
  3. 包括角xeditable到项目 
    <link href="bower_components/angular-xeditable/dist/css/xeditable.css" rel="stylesheet">
<script src="bower_components/angular-xeditable/dist/js/xeditable.js"></script>
  4. 定义应用程序并添加“xeditable角”的依赖 
    <html ng-app="app">
    var app = angular.module("app", ["xeditable"]);
  5. 在app.run设置主题: 
    app.run(function(editableOptions) {
  editableOptions.theme = 'bs3'; // bootstrap3主题。也可以 'bs2', 'default'
});
  6. 标记元素都应该是可编辑的 
    <div ng-controller="Ctrl">
  <a href="#" editable-text="user.name">{{ user.name || "empty" }}</a>
</div>
  7. 定义控制器 
    app.controller('Ctrl', function($scope) {
  $scope.user = {
    name: 'awesome user'
  };  
});
  8. Enjoy!
    查看启动模板下载启动 zip

选择本地

演示

jsFiddle
{{ showStatus() }} 
{{ debug["select-local"] | json }}

创建可编辑的选择（下拉）是集editable-select 指向模型的属性。 通过你应该定义下拉选项 e-ng-options 属性 的作品像正常的角 ng-options 但被标 <select> 从原始的元素。 要设置列表中的默认选项添加e-placeholder 属性.

html

<div ng-controller="SelectLocalCtrl">
  <a href="#" editable-select="user.status" e-ng-options="s.value as s.text for s in statuses">
    {{ showStatus() }}
  </a>
</div>

controller.js

app.controller('SelectLocalCtrl', function($scope, $filter) {
  $scope.user = {
    status: 2
  }; 

  $scope.statuses = [
    {value: 1, text: 'status1'},
    {value: 2, text: 'status2'},
    {value: 3, text: 'status3'},
    {value: 4, text: 'status4'}
  ]; 

  $scope.showStatus = function() {
    var selected = $filter('filter')($scope.statuses, {value: $scope.user.status});
    return ($scope.user.status && selected.length) ? selected[0].text : 'Not set';
  };
});

选择远程

演示

jsFiddle
{{ user.groupName || 'not set' }} 
{{ debug["select-remote"] | json }}

要加载从远程网址中选择的选项，你应该定义 onshow 属性 指向范围函数。 函数的结果应该是一个为HTTP的承诺，它可以禁用元素同时加载.

html

<div ng-controller="SelectRemoteCtrl">
  <a href="#" editable-select="user.group" onshow="loadGroups()" e-ng-options="g.id as g.text for g in groups">
    {{ user.groupName || 'not set' }}
  </a>
</div>

controller.js

app.controller('SelectRemoteCtrl', function($scope, $filter, $http) {
  $scope.user = {
    group: 4,
    groupName: 'admin' // 原始值
  }; 

  $scope.groups = [];

  $scope.loadGroups = function() {
    return $scope.groups.length ? null : $http.get('/groups').success(function(data) {
      $scope.groups = data;
    });
  };

  $scope.$watch('user.group', function(newVal, oldVal) {
    if (newVal !== oldVal) {
      var selected = $filter('filter')($scope.groups, {id: $scope.user.group});
      $scope.user.groupName = selected.length ? selected[0].text : null;
    }
  });
});

清单

演示

{{ showStatus() }} 
{{ debug["checklist"] | json }}

创建列表框的使用 editable-checklist 指向模型的属性. 你也应该定义 e-ng-options 属性设置值和显示项目.

请注意, 你应该包括 checklist-model directive为你的应用程序: var app = angular.module("app", [..., "checklist-model"]);.

默认情况下，复选框对齐 horizontally. 对齐 vertically 只需添加以下 CSS:

.editable-checklist label {
  display: block;
}

html

<div ng-controller="ChecklistCtrl">
  <a href="#" editable-checklist="user.status" e-ng-options="s.value as s.text for s in statuses">
    {{ showStatus() }}
  </a>
</div>

controller.js

app.controller('ChecklistCtrl', function($scope, $filter) {
  $scope.user = {
    status: [2, 3]
  }; 

  $scope.statuses = [
    {value: 1, text: 'status1'},
    {value: 2, text: 'status2'},
    {value: 3, text: 'status3'}
  ]; 

  $scope.showStatus = function() {
    var selected = [];
    angular.forEach($scope.statuses, function(s) { 
      if ($scope.user.status.indexOf(s.value) >= 0) {
        selected.push(s.text);
      }
    });
    return selected.length ? selected.join(', ') : 'Not set';
  };

});

日期

演示

View Bootstrap 2 jsFiddle
{{ (user.dob | date:"dd/MM/yyyy") || 'empty' }} 
{{ debug["bsdate"] | json }}

日期控制通过 Angular-ui bootstrap datepicker.
你应该包括 额外的 ui-bootstrap-tpls.min.js:

<script src="https://cdnjs.c2cbc.com/ajax/libs/angular-ui-bootstrap/1.3.1/ui-bootstrap-tpls.min.js"></script>

添加 ui.bootstrap 作为模块依赖：

var app = angular.module("app", ["xeditable", "ui.bootstrap"]);

并设置 editable-bsdate 在编辑元件属性。 使输入字段的只读和力要选择的日期从弹出，添加 e-readonly="true" 属性.
添加 e-ng-change 属性来调用一个函数时的DatePicker改变价值.
隐藏在输入字段单击日历弹出日历按钮，显示设置 e-show-calendar-button 属性 to false.
其他参数可以通过e-* syntax, e.g. e-datepicker-popup="dd-MMMM-yyyy".

html

<div ng-controller="BsdateCtrl">
  <a href="#" editable-bsdate="user.dob" e-is-open="opened.$data" e-ng-click="open($event,'$data')" e-datepicker-popup="dd-MMMM-yyyy">
    {{ (user.dob | date:"dd/MM/yyyy") || 'empty' }}
  </a>
</div>

controller.js

app.controller('BsdateCtrl', function($scope) {
	$scope.user = {
		dob: new Date(1984, 4, 15)
	};

	$scope.opened = {};

	$scope.open = function($event, elementOpened) {
		$event.preventDefault();
		$event.stopPropagation();

		$scope.opened[elementOpened] = !$scope.opened[elementOpened];
	};
});

  content: '\e113';
}
.editable-bstime .editable-input i.icon-chevron-down:before {
  content: '\e114';
}
.editable-bstime .editable-input i.icon-chevron-up,
.editable-bstime .editable-input i.icon-chevron-down {
  position: relative;
  top: 1px;
  display: inline-block;
  font-family: 'Glyphicons Halflings';
  -webkit-font-smoothing: antialiased;
  font-style: normal;
  font-weight: normal;
  line-height: 1;
}

通过onbeforesave提交

演示

jsFiddle
{{ user.name || 'empty' }} 
{{ debug["onbeforesave"] | json }}

在服务器上提交数据的一种方法是定义 onbeforesave 属性指向某一范围的方法。 有用的当你需要在服务器上发送数据，然后才更新本地模型（例如. $scope.user). 新的值可以通过 $data 参数（如. <a ... onbeforesave="updateUser($data)">).

最主要的是，只有当方法返回时，本地模型将被更新 true或undefined （或承诺解决 true/undefined). 一般情况下，有3种情况取决于结果类型:

  • true 或 undefined: 成功。局部模型将自动更新，表单将关闭.
  • false: 成功。但将局部模型 not被更新和形式将关闭。当您想手动更新本地模型（例如服务器更改值）时，非常有用.
  • string: Error.Local 模型 将 not 被更新，形式将 not关闭，字符串将被显示为错误消息. 用于验证和处理错误.

html

<div ng-controller="OnbeforesaveCtrl">
  <a href="#" editable-text="user.name" onbeforesave="updateUser($data)">
    {{ user.name || 'empty' }}
  </a>
</div>

controller.js

app.controller('OnbeforesaveCtrl', function($scope, $http) {
  $scope.user = {
    id: 1,
    name: 'awesome user'
  };

  $scope.updateUser = function(data) {
    return $http.post('/updateUser', {id: $scope.user.id, name: data});
  };
});

在服务器上提交数据的另一种方法是定义 onaftersave 属性指向某一范围的方法。 当您需要先更新本地模型，然后将其发送到服务器时，非常有用。 有没有输入参数作为数据已经在当地的模型.

此方法的结果类型可以如下:

  • 但任何事情string：成功，表格将被关闭
  • string：错误，窗体将不会关闭，字符串将显示为错误信息

请注意，您既可以使用onbeforesave验证和onaftersave保存数据。

  </a>
</div>

app.controller('OnaftersaveCtrl', function($scope, $http) {
  $scope.user = {
    id: 1,
    name: 'awesome user'
  };

  $scope.updateUser = function() {
    return $http.post('/updateUser', $scope.user);
  };

可编辑的形式

演示

jsFiddle
用户名： {{ user.name || 'empty' }}
状态： {{ (statuses | filter:{value: user.status})[0].text || 'Not set' }}
Group: {{ showGroup() }}

要显示几个编辑元素结合在一起，并在一次你应该换到他们提交<form editable-form name="myform" ...>的标签。该name表格的属性将创建范围的变量（正常角度行为）和editable-form属性将添加一些方法，以该变量：

  • $show()
  • $cancel()
  • $visible
  • $waiting

用它来 切换形式的可编辑状态。例如，你可以调用myform.$show()。

可编辑的形式支持其他3个属性：

  • OnShow中：调用时显示形式
  • onbeforesave：呼吁之前本土车型更新提交
  • onaftersave：呼吁后局部模型提交更新

他们的工作几乎相同个别编辑的元素。使用它，而不是ng-submit / submit得到了保存过程更多的控制。

当您提交可编辑的表单将执行以下步骤：

  1. 呼儿的 onbeforesave
  2. 调用方式的 onbeforesave
  3. 数据写入本地模型（如$scope.user）
  4. 调用方式的 onaftersave
  5. 呼儿的 onaftersave

任何onbeforesave / onaftersave可以被省略了所以在简单的情况下，你只将数据写入本地模式。

但在更复杂的情况下，它变成有用：

如果你需要单独的可编辑元素的验证，那么你应该定义onbeforesave。特别编辑元素的
孩子的结果onbeforesave是下一步重要的：

  • string：提交将被取消，表格将保持开通，字符串将显示为错误信息
  • not string：提交将继续

如果您需要发送服务器上的数据之前写入局部模型，那么你应该定义表单onbeforesave，
表单的结果onbeforesave是下一步重要的：

  • true或undefined：局部模型将被更新和形式将调用onaftersave
  • false：局部模型将不会被更新和形式将正好接近（如更新局部模型自己）
  • string：局部模型将不会被更新和表单不会关闭（如服务器错误）

如果您需要发送的服务器数据后写入到本地模式，那么你应该定义表单onaftersave，
表单的结果onaftersave也是下一步重要的：

  • string：窗体不会关闭（如服务器错误）
  • not string：表格将被关闭

通常你应该定义onbeforesave为子元素进行验证和onaftersave对整个表单发送到服务器上的数据。

注： e-required不会工作，因为HTML5的验证只能当提交表单与一个提交按钮，并editable-form通过脚本提交。

请看看例子。

html

<div ng-controller="EditableFormCtrl">
  <form editable-form name="editableForm" onaftersave="saveUser()">
    <div>
      <!-- editable username (text with validation) -->
      <span class="title">User name: </span>
      <span editable-text="user.name" e-name="name" onbeforesave="checkName($data)" e-required>{{ user.name || 'empty' }}</span>
    </div> 

    <div>
      <!-- editable status (select-local) -->
      <span class="title">Status: </span>
      <span editable-select="user.status" e-name="status" e-ng-options="s.value as s.text for s in statuses">
        {{ (statuses | filter:{value: user.status})[0].text || 'Not set' }}
      </span>
    </div>  

    <div>
      <!-- editable group (select-remote) -->
      <span class="title">Group: </span>
      <span editable-select="user.group" e-name="group" onshow="loadGroups()" e-ng-options="g.id as g.text for g in groups">
        {{ showGroup() }}
      </span>
    </div>

    <div class="buttons">
      <!-- button to show form -->
      <button type="button" class="btn btn-default" ng-click="editableForm.$show()" ng-show="!editableForm.$visible">
        Edit
      </button>
      <!-- buttons to submit / cancel form -->
      <span ng-show="editableForm.$visible">
        <button type="submit" class="btn btn-primary" ng-disabled="editableForm.$waiting">
          Save
        </button>
        <button type="button" class="btn btn-default" ng-disabled="editableForm.$waiting" ng-click="editableForm.$cancel()">
          Cancel
        </button>
      </span>
    </div>
  </form>  
</div>

controller.js

app.controller('EditableFormCtrl', function($scope, $filter, $http) {
  $scope.user = {
    id: 1,
    name: 'awesome user',
    status: 2,
    group: 4,
    groupName: 'admin'
  }; 

  $scope.statuses = [
    {value: 1, text: 'status1'},
    {value: 2, text: 'status2'},
    {value: 3, text: 'status3'},
    {value: 4, text: 'status4'}
  ]; 

  $scope.groups = [];
  $scope.loadGroups = function() {
    return $scope.groups.length ? null : $http.get('/groups').success(function(data) {
      $scope.groups = data;
    });
  };

  $scope.showGroup = function() {
    if($scope.groups.length) {
      var selected = $filter('filter')($scope.groups, {id: $scope.user.group});
      return selected.length ? selected[0].text : 'Not set';
    } else {
      return $scope.user.groupName;
    }
  };

  $scope.checkName = function(data) {
    if (data !== 'awesome' && data !== 'error') {
      return "Username should be `awesome` or `error`";
    }
  };

  $scope.saveUser = function() {
    // $scope.user already updated!
    return $http.post('/saveUser', $scope.user).error(function(err) {
      if(err.field && err.msg) {
        // err like {field: "name", msg: "Server-side error for this username!"} 
        $scope.editableForm.$setError(err.field, err.msg);
      } else { 
        // unknown error
        $scope.editableForm.$setError('name', 'Unknown error!');
      }
    });
  };
});

  $scope.users = [
    {id: 1, name: 'awesome user1', status: 2, group: 4, groupName: 'admin'},
    {id: 2, name: 'awesome user2', status: undefined, group: 3, groupName: 'vip'},
    {id: 3, name: 'awesome user3', status: 2, group: null}
  ]; 

  $scope.statuses = [
    {value: 1, text: 'status1'},
    {value: 2, text: 'status2'},
    {value: 3, text: 'status3'},
    {value: 4, text: 'status4'}
  ]; 

  $scope.groups = [];
  $scope.loadGroups = function() {
    return $scope.groups.length ? null : $http.get('/groups').success(function(data) {
      $scope.groups = data;
    });
  };

  $scope.showGroup = function(user) {
    if(user.group && $scope.groups.length) {
      var selected = $filter('filter')($scope.groups, {id: user.group});
      return selected.length ? selected[0].text : 'Not set';
    } else {
      return user.groupName || 'Not set';
    }
  };

  $scope.showStatus = function(user) {
    var selected = [];
    if(user.status) {
      selected = $filter('filter')($scope.statuses, {value: user.status});
    }
    return selected.length ? selected[0].text : 'Not set';

    if (id === 2 && data !== 'awesome') {
      return "Username 2 should be `awesome`";

  };

    //$scope.user not updated yet
    angular.extend(data, {id: id});

 // 移除用户

    $scope.users.splice(index, 1);
  };

  // 添加用户

    $scope.inserted = {
      id: $scope.users.length+1,
      name: '',
      status: null,
      group: null 
    };
    $scope.users.push($scope.inserted);
  };

  };

  $scope.checkName = function(data) {
    if (data !== 'awesome') {
      return "Username should be `awesome`";
    }
  };

  $scope.saveColumn = function(column) {
    var results = [];
    angular.forEach($scope.users, function(user) {
      results.push($http.post('/saveColumn', {column: column, value: user[column], id: user.id}));
    })
    return $q.all(results);
  };

});

        </td>

          <!-- editable group (select-remote) -->

controller.js

app.controller('EditableTableCtrl', function($scope, $filter, $http, $q) {
  $scope.users = [
    {id: 1, name: 'awesome user1', status: 2, group: 4, groupName: 'admin'},
    {id: 2, name: 'awesome user2', status: undefined, group: 3, groupName: 'vip'},
    {id: 3, name: 'awesome user3', status: 2, group: null}
  ]; 

  $scope.statuses = [
    {value: 1, text: 'status1'},
    {value: 2, text: 'status2'},
    {value: 3, text: 'status3'},
    {value: 4, text: 'status4'}
  ]; 

  $scope.groups = [];
  $scope.loadGroups = function() {
    return $scope.groups.length ? null : $http.get('/groups').success(function(data) {
      $scope.groups = data;
    });
  };

  $scope.showGroup = function(user) {
    if(user.group && $scope.groups.length) {
      var selected = $filter('filter')($scope.groups, {id: user.group});
      return selected.length ? selected[0].text : 'Not set';
    } else {
      return user.groupName || 'Not set';
    }
  };

  $scope.showStatus = function(user) {
    var selected = [];
    if(user.status) {
      selected = $filter('filter')($scope.statuses, {value: user.status});
    }
    return selected.length ? selected[0].text : 'Not set';
  };

  $scope.checkName = function(data, id) {
    if (id === 2 && data !== 'awesome') {
      return "Username 2 should be `awesome`";
    }
  };

  //过滤用户显示
  $scope.filterUser = function(user) {
    return user.isDeleted !== true;
  };

  // 标记用户已删除
  $scope.deleteUser = function(id) {
    var filtered = $filter('filter')($scope.users, {id: id});
    if (filtered.length) {
      filtered[0].isDeleted = true;
    }
  };

  // 添加用户
  $scope.addUser = function() {
    $scope.users.push({
      id: $scope.users.length+1,
      name: '',
      status: null,
      group: null,
      isNew: true
    });
  };

  // 取消所有更改
  $scope.cancel = function() {
    for (var i = $scope.users.length; i--;) {
      var user = $scope.users[i];    
      // undelete
      if (user.isDeleted) {
        delete user.isDeleted;
      }
      // remove new 
      if (user.isNew) {
        $scope.users.splice(i, 1);
      }      
    };
  };

   // 保存编辑
  $scope.saveTable = function() {
    var results = [];
    for (var i = $scope.users.length; i--;) {
      var user = $scope.users[i];
      //实际删除用户
      if (user.isDeleted) {
        $scope.users.splice(i, 1);
      }
      //标记为不新
      if (user.isNew) {
        user.isNew = false;
      }

      // 发送服务器
      results.push($http.post('/saveUser', user));      
    }

    return $q.all(results);
  };
});

  editableOptions.theme = 'bs3';

Set theme in app.run:

app.run(function(editableOptions) {
  editableOptions.theme = 'bs2';
});

Default

No additional CSS required.
You can customize theme in app.run by overwriting properties:

app.run(function(editableOptions, editableThemes) {
  // set `default` theme
  editableOptions.theme = 'default';
  
  // overwrite submit button template
  editableThemes['default'].submitTpl = '<button type="submit">ok</button>';
});

属性

属性可以为任何拥有元素定义editable-xxx指令：

<a href="#" editable-text="user.name" [attributes]> {{user.name | "empty"}} </a>
Name 类型 描述
blur 串

操作时控制损失的焦点。值：cancel|submit|ignore。只有单可编辑元素的感觉。否则，如果控制表单的一部分-你应该设置blur形式，而不是单个元素。
buttons 串

是否显示确定/取消按钮。值：right|no。如果设置为no控制值时会自动改变提交。
如果表单按钮的一部分将永远不会被显示的控制。
e-* 任何

与定义的属性e-*前缀从控制原始元素会自动转移。
例如，如果你设置<span editable-text="user.name" e-style="width: 100px">然后输入将显示为<input style="width: 100px">。
见演示。
onaftersave 方法

在所谓价值被保存到模型后提交。
见演示。
onbeforesave 方法

在调用之前的值保存到模型提交。
见演示。
oncancel 方法

当控制被取消调用。
onhide 方法

当控制隐藏后都保存或取消调用。
onshow 方法

显示控制时调用。
见演示。

所有的孩子后调用onbeforesave回调，但保存表单值模型前。

参见编辑形式演示的详细信息。

当表单被取消调用。
onhide 方法

当调用隐藏表单后都保存或取消。
onshow 方法

显示窗体时调用。
shown 布尔

无论是最初的形式呈现在显示状态。
